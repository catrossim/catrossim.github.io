  • ssh

目标

  1. 远程服务器(R)创建用户test，并对test创建用户目录、指定shell、初始化密码。
  2. 远程服务器(R)禁用root用户登陆，可用test进行免密码登陆。
  3. 本地服务器(L)通过ssh登陆到远程服务器(R)时不需要输入密码；

    操作环境

  • Local Server（L）:

    • os: OSX 10.11.4

  • Remote Server（R）:

L部分

概要

本地服务器需要做的事——生成密钥，私钥自己保管，公钥上传到远程服务器R。

生成密钥

执行命令ssh-keygen，提供交互式操作，默认生成长度为2048bits的RSA编码串。需要输入文件保存路径

执行结果是在/home/chenjh/.ssh/下生成私钥id_rsa_plus和公钥id_rsa_plus.pub。

由于需要管理多台服务器的密钥，这里并没有使用默认的密钥保存路径/home/chenjh/.ssh/id_rsa。习惯上，我会在id_rsa后面加上服务器名称标识。例如这里是id_rsa_plus。

在ssh到远程服务器R的过程中，默认是查找/home/chenjh/.ssh/id_rsa进行匹配。但这里私钥的文件路径是/home/chenjh/.ssh/id_rsa_plus，因此还需要进行一些处理。

更改验证路径

修改位置在文件~/.ssh/config，假如没有这个文件，需要先人工创建

vim ~/.ssh/config。

注意每行有空格。

Host <SHORT_SERVER_NAME>定义一个服务器昵称，之后ssh的时候不用输入整个服务器地址，只要昵称就好。例如ssh test@plus。

Hostname <IP_OR_HOST_NAME>对应是服务器真实的ip地址或者主机名。

IdentitiesOnly yes表示只使用IdentityFile定义的文件。假如在ssh-agent有多个私钥对应，那么在这里也只是用IdentityFile定义的。

IdentityFile <PRIVATE_KEY_PATH>指定私钥位置。

在需要管理多组密钥密钥时，建议使用这种方式配置。
